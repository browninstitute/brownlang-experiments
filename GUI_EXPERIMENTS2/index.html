Brown-Tow Programming Project
GUI Expriments Part II

After a few experiments on the language part of the project (see here), and two others on how to adapt literary notation to programming interface (see here and here), I wanted to go deeper on the "programmy" part. To put it differently, what tools could be much more useful for novice programmers, regardless of the langage used?

I made a few in-browser experiments to simulate what a few of these tools would look like with Python programs. In each example, I used a snippet of code, assuming it was typed by the user. Generally, there is a or a located next to this snippet. By clicking on each of them, the user can see the code in a different, often more tangible way.

Only 6 experiments are fully finished now. I have 3 more in my head. However, when it comes to make small tools to make programming more tangible, the field seems almost infinite.
