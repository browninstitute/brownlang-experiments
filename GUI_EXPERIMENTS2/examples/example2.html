2 - An object inspector

A user should be able to inspect the content of objects easily. This is, of course, important for debugging. But it is also important for the novice to learn and become accustomed to the different data types. In particular, if the GUI displays the different data types in a graphically specific way, it could make the data more tangible.

Now with more complex data structures. Let us imagine that the user uses a table-like data structure. The table plays the role of data frame to store information about TV series. This table comprises a large amount of rows. Each row is organized the following way:
(0) A pair:
      (Key) A string for the name of the series
      (Value) A list with string of names of the creators of the series
(1) A float comprising the rating
(2) A tree that lays out the seasons and the episodes.
      There are Season objects, which are comprised of an ID (integer), a name (string), and a rating(float).
      There are Episode objects, which are comprised of an ID (integer), a name (string), a rating (float), and a duration (time).
